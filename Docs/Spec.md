# エンジン回転数PID制御デモ 仕様書

## 1. 概要

エンジンの回転数を目標値に自動制御するデモアプリケーション。
PID制御器が「アクセル開度UP」「アクセル開度DOWN」ボタンを自動的に操作し、エンジン回転数を目標値に収束させる様子をビジュアルで確認できる。

## 2. 技術スタック

| 項目 | 選定 |
|------|------|
| 言語 | C# |
| フレームワーク | Avalonia UI |
| ターゲット | .NET 8 |
| アーキテクチャ | MVVM |

## 3. エンジンシミュレーションモデル

### 3.1 パラメータ

| パラメータ | 範囲 | 単位 |
|-----------|------|------|
| アクセル開度 | 0 ~ 100 | % |
| エンジン回転数 | 0 ~ 3000 | rpm |
| 回転数検知遅延 | 約2秒 | - |

### 3.2 エンジン挙動モデル

- アクセル開度に応じた定常回転数が存在する（**非線形特性**）
- 開度変更後、実際の回転数は一次遅れ系で定常値に追従する
- 回転数の検知（センサー値の更新）には約2秒の遅延がある

#### 非線形トルクカーブ（開度 → 定常回転数）

低開度域では回転数の上昇が緩やかで、中開度域で急峻に立ち上がり、高開度域で飽和する特性とする。

| アクセル開度 (%) | 定常回転数 (rpm) | 備考 |
|-----------------|-----------------|------|
| 0 | 0 | エンジン停止 |
| 10 | 150 | アイドル付近 |
| 20 | 400 | 低回転域 |
| 30 | 750 | |
| 40 | 1150 | |
| 50 | 1550 | 中回転域 |
| 60 | 1900 | |
| 70 | 2200 | |
| 80 | 2500 | 高回転域（飽和傾向） |
| 90 | 2750 | |
| 100 | 3000 | 最大回転数 |

※ 中間値はスプライン補間またはルックアップテーブル+線形補間で算出する。
※ 実装時に曲線の形状は調整可能とする。

### 3.3 アクセル開度の変化モデル

- 「UPボタン押下中」: 開度が一定速度で増加（例: 毎秒10%）
- 「DOWNボタン押下中」: 開度が一定速度で減少（例: 毎秒10%）
- ボタンを離すと開度は現在値で保持される
- 開度が0%に達した場合: それ以上下がらず0%で停止
- 開度が100%に達した場合: それ以上上がらず100%で停止

## 4. PID制御

### 4.1 制御方式

- PID制御器は目標回転数と検知回転数（2秒遅延あり）の偏差を入力とする
- PID出力が正の場合: 「UPボタン」を自動で押す（開度を上げる）
- PID出力が負の場合: 「DOWNボタン」を自動で押す（開度を下げる）
- PID出力が0付近（不感帯内）の場合: どちらのボタンも押さない
- 制御周期: 100ms

### 4.2 PIDゲイン

| ゲイン | 初期値 | 調整範囲 |
|--------|--------|----------|
| Kp（比例） | 0.5 | 0.0 ~ 5.0 |
| Ki（積分） | 0.1 | 0.0 ~ 2.0 |
| Kd（微分） | 0.05 | 0.0 ~ 2.0 |

※ 初期値は仮。実装後にチューニングして適正値を決定する。

### 4.3 積分ワインドアップ対策

- 積分項にアンチワインドアップを実装する
- 積分値の上下限をクランプする

### 5.2 表示要素

| 要素 | 表示形式 | 説明 |
|------|----------|------|
| 目標回転数 | スライダー + 数値 | 0~3000rpm。ユーザーがスライダーで設定 |
| 検知回転数 | プログレスバー + 数値 | センサー検知値（2秒遅延あり） |
| アクセル開度 | プログレスバー + 数値 | 現在のアクセル開度 0~100% |
| ボタン状態 | ボタンUI（ハイライト） | UP/DOWNどちらが押されているか視覚表示 |

| リアルタイムグラフ | 折れ線グラフ | 目標回転数（青）と検知回転数（赤）の時系列推移を表示。横軸は時間（直近30秒程度をスクロール表示） |

### 5.3 操作モード

#### PID制御ONモード
- PID制御器がUP/DOWNボタンを自動操作する
- ボタンが自動で押されている様子がUIに反映される（ハイライト表示）
- 手動操作ボタンは無効化（グレーアウト）
- 目標回転数のスライダーは操作可能

#### PID制御OFFモード（手動モード）
- ユーザーが手動でUP/DOWNボタンを長押し操作する
- PIDゲイン調整スライダーは操作可能（次回ON時に反映）

## 6. クラス設計（概要）

### 6.1 モデル層

- **EngineSimulator**: エンジンの物理シミュレーション
  - アクセル開度 → 定常回転数の変換
  - 一次遅れ系による回転数応答
  - センサー遅延（2秒）のシミュレーション
- **ThrottleActuator**: アクセル開度アクチュエータ
  - UP/DOWN入力に応じた開度変化
  - 上下限クランプ
- **PidController**: PID制御器
  - 偏差計算、P/I/D各項の算出
  - アンチワインドアップ
  - UP/DOWNボタン操作の出力

### 6.2 ViewModel層

- **MainViewModel**: メインウィンドウのViewModel
  - 各モデルの保持と制御ループの管理
  - UIバインディング用プロパティ

### 6.3 View層

- **MainWindow**: メインウィンドウ（Avalonia XAML）

## 7. 制御ループのタイミング

```
[100msタイマー]
  ├─ センサーから検知回転数を取得（2秒遅延付き）
  ├─ PID演算（目標 - 検知 = 偏差）
  ├─ PID出力に応じてUP/DOWNボタンを操作
  └─ UIを更新

[シミュレーション内部: 10ms刻み]
  ├─ アクセル開度の更新（ボタン状態に応じて増減）
  ├─ エンジン回転数の更新（一次遅れ系）
  └─ センサーバッファへの回転数記録（2秒遅延用）
```

## 8. リアルタイムグラフ

### 8.1 表示内容

| 系列 | 色 | 説明 |
|------|-----|------|
| 目標回転数 | 青 | ユーザーが設定した目標値 |
| 検知回転数 | 赤 | センサーから取得した値（2秒遅延あり） |
| アクセル開度 | 緑（副軸） | 現在のアクセル開度（0~100%、右側Y軸） |

### 8.2 グラフ仕様

- 横軸: 時間（直近30秒をスクロール表示）
- 左Y軸: 回転数（0~3000rpm）
- 右Y軸: アクセル開度（0~100%）
- 更新間隔: 100ms（制御ループと同期）
- Avaloniaのカスタム描画（DrawingContext）またはサードパーティライブラリで実装
